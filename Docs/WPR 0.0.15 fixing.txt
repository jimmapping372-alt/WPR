Why this fixes the white screen
•	The white screen on Android is commonly caused by Avalonia not being initialized or the activity that should host the UI not being properly configured before the framework tries to render. Previously CustomizeAppBuilder returned a noop/default and activity-level initialization (activity result registration) occurred too early in the constructor; that can prevent Avalonia from being configured and the MainActivity from starting the UI, producing a white/blank screen. Calling the base CustomizeAppBuilder(...) and applying Avalonia options ensures the framework is initialized properly. Launch flags on the splash activity avoid activity duplication/state issues that can also result in no UI being shown.
