Corrections and causes

  - MainActivity.cs
  - If you need ReactiveUI on Android, you can add Avalonia.ReactiveUI in WPR.UI.Android.csproj and return UseReactiveUI.
Warnings and recommendations

- NU1701 by WindowsAPICodePack-* and HtmlAgilityPack.CssSelectors:
  - The packages are assembled under .NET Framework, used in projects with net8.0; warnings are acceptable, but it is worth migrating over time:
    - For Windows: replace WindowsAPICodePack with modern analogues (WinUI/WinRT, Vanara).
    - For general HTML: consider AngleSharp instead of HtmlAgilityPack.CssSelectors.
- MSB3270 (architecture mismatch FNA.dll AMD64 vs MSIL project):
- Risk of runtime errors, but the build goes through. It is recommended to align the project architecture to x64 or use AnyCPU-compatible builds.
