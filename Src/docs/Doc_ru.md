# Overview

Эмулятор игр Windows Phone 7 под названием **WPR** изначально реализован на платформе **Avalonia**. Задача — портировать WPR на другие UI-стэки: сначала **Xamarin.Forms**, затем **.NET MAUI**.

Особенность текущей реализации WPR на Avalonia заключается в том, что в ней разрешено выполнять модификацию динамических библиотек (assembly) в **runtime**. Для целей эмуляции используется механизм «инсталляции игры»: распаковка XAP-файла и модификация его библиотек с помощью **Mono.Cecil**.

При переносе на **Xamarin.Forms** (особенно при таргете **UWP**) возникают ограничения «песочницы» платформы: модификация библиотек во время выполнения приложения запрещена. Поэтому необходимо изменить архитектуру: перенести этап модификации библиотек из runtime в отдельный оффлайн-конвертер.


# Goals

1. Создать оффлайн-конвертер XAP-файлов в новый формат **WPR**:
   - входной формат: `.xap` (архив с исходными библиотеками игры);
   - выходной формат: `.wpr` (архив, содержащий уже пропатченные через Mono.Cecil библиотеки и необходимые метаданные).

2. Исследовать существующий солюшн **WPR** на Avalonia и найти реализацию «механизма инсталляции игры»:
   - распаковка XAP-файла;
   - поиск и загрузка сборок игры;
   - патчинг сборок с помощью **Mono.Cecil**;
   - текущее место, где это вызывается и как интегрируется с эмулятором.

3. На основе обнаруженной логики подготовить новый проект **XapToWPR** (Avalonia-приложение) внутри солюшна **WPR.sln**:
   - проект `XapToWPR.csproj` должен быть добавлен в `WPR.sln`;
   - конвертер реализует тот же патчинг, который сейчас делается в runtime, но выполняет его заранее и сохраняет результат в `.wpr`.

4. Обеспечить совместимость с будущими портами на **Xamarin.Forms** и **MAUI**:
   - чтобы на этих платформах эмулятор работал с уже подготовленными `.wpr`-файлами без необходимости модификации сборок во время выполнения.

5. Соблюдать требования к сборке:
   - сборку всего солюшна **WPR.sln** выполнять через **MSBuild**, а не `dotnet build`:
     - `MSBuild.exe "WPR.sln" /p:Configuration=Debug /p:Platform=x64 /t:Build`;
     - при необходимости (при ошибках/отсутствии пакетов) предварительно:
       `MSBuild.exe "WPR.sln" /t:Restore`.
